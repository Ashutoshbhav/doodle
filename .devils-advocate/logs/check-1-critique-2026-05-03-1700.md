## Failing criteria — fixes

### CRITICAL (build-blocking)

1. **`PatchWall.tsx:31` and `Founders.tsx:12` — `Icon` type is not exported from `@phosphor-icons/react/dist/ssr`.**
   The `Icon` type lives in the package root only. The SSR subpath only re-exports the per-icon components.
   **Fix (option A — preferred):** Import the type from the package root, components from SSR.
   ```tsx
   import { Star, Heart, /* ... */ } from "@phosphor-icons/react/dist/ssr";
   import type { Icon } from "@phosphor-icons/react";
   ```
   **Fix (option B):** Drop the explicit `Icon` annotation and use `React.ComponentType<{ weight?: string; size?: number; className?: string }>` — uglier.

2. **`Hero.tsx:53` — dynamic Tailwind class `bg-${c}` is purged.**
   Tailwind v4 JIT scans for **complete** class strings; `bg-${c}` is not detected, so `bg-doodle-orange` etc. won't be in the CSS bundle and the three avatar dots render colourless.
   **Fix:**
   ```tsx
   const DOT_BG: Record<typeof c, string> = {
     "doodle-orange": "bg-doodle-orange",
     "doodle-blue": "bg-doodle-blue",
     "doodle-yellow": "bg-doodle-yellow",
   };
   // ...
   className={`inline-block h-6 w-6 rounded-full ${DOT_BG[c]} border-2 border-dashed border-doodle-stitch`}
   ```

### IMPORTANT (lint errors / real code smell)

3. **`Footer.tsx:111` — replace `dangerouslySetInnerHTML` with plain text.**
   The only reason it's there is to render `&` in `"Care & washing"`. JSX renders `&` literally already.
   **Fix:**
   ```tsx
   { href: "#", label: "Care & washing" },
   // ...
   <Link href={l.href} className="...">{l.label}</Link>
   ```

4. **`WhyDoodle.tsx:173` — unescaped apostrophe.**
   ```tsx
   [placeholder copy &mdash; rewrite in Ash&rsquo;s voice]
   ```

5. **`PatchScrubber.tsx:303` — `setState` in effect body triggers cascading render (React 19 lint error).**
   **Fix:** initialise lazily and only subscribe in the effect.
   ```tsx
   const [reduced, setReduced] = React.useState(() => false);
   React.useEffect(() => {
     const mq = window.matchMedia("(prefers-reduced-motion: reduce)");
     setReduced(mq.matches);            // outside the synchronous-render path now
     const onChange = (e: MediaQueryListEvent) => setReduced(e.matches);
     mq.addEventListener("change", onChange);
     return () => mq.removeEventListener("change", onChange);
   }, []);
   ```
   Or use `useSyncExternalStore` for proper SSR/initial-paint correctness.

### MINOR (warnings + brittleness)

6. **Drop unused imports.** `EnvelopeSimple` from `DualCTA.tsx`, `Storefront` from `FindUsOffline.tsx`.

7. **`MarkerHeading.tsx` — silent failure if `markerWord` not in the chosen slot.** Add a `console.warn` in dev or fall back to wrapping the whole word group. Also the `split` logic should be cached and handle multi-occurrence safely.

8. **`PatchScrubber.tsx:241-250` — soft tee hexes are off-palette.** Either add them as proper `--color-doodle-*-soft` tokens, or document them as intentional UX tints not part of the brand palette.
